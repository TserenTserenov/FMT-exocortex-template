Выполни сценарий Day Plan для агента Стратег.

## Контекст

- **HUB (личные планы):** ~/Github/DS-strategy/current/
- **SPOKE (планы репо):** ~/Github/*/WORKPLAN.md
- **MEMORY:** MEMORY.md (автозагрузка)

## Именование файлов в current/

```
DS-strategy/
├── current/
│   ├── WeekPlan W{N} YYYY-MM-DD.md   # план недели (Пн дата)
│   ├── WeekReport W{N} YYYY-MM-DD.md # отчёт недели (Вс дата)
│   └── DayPlan YYYY-MM-DD.md         # план дня (текущая дата)
├── archive/                           # старые файлы
```

В `current/` — только актуальные файлы. Старые перемещаются в `DS-strategy/archive/`.

## Алгоритм

### 1. Итоги вчера (Стратег собирает сам)

**Стратег ОБЯЗАН** собрать коммиты за вчерашний день самостоятельно:

```bash
# Для КАЖДОГО репо в ~/Github/:
git -C ~/Github/<repo> log --since="yesterday 00:00" --until="today 00:00" --oneline --no-merges
```

- Пройди по ВСЕМ репозиториям в `~/Github/`
- Сгруппируй коммиты по репозиториям
- Сопоставь коммиты с РП из недельного плана
- Определи статус каждого затронутого РП

### 2. Обновить WeekPlan по итогам вчера

- Пометь завершённые РП как **done**
- Обнови статусы partial с описанием прогресса
- **НЕ удаляй** ничего — только помечай и дописывай

### 3. Контекст недели

- Загрузи обновлённый WeekPlan
- Рассчитай прогресс: done/total РП, оставшийся бюджет

### 4. План на сегодня

- Выбери 2-4 РП из недельного плана
- Учти carry-over со вчера
- Ограничь по дневному бюджету (4-6h)

### 5. Рекомендация

- Предложи с чего начать и почему
- Укажи входные материалы

### 6. Сохранение

1. Если в `current/` есть предыдущий `DayPlan *.md` — перемести его в `archive/`
2. Создай новый файл: `current/DayPlan YYYY-MM-DD.md`
3. Закоммить в DS-strategy

Результат: обновлённый WeekPlan + DayPlan в `current/`.

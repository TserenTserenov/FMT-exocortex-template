Выполни сценарий «Сессия стратегирования» для агента Стратег.

## Контекст

- **HUB (личные планы):** ~/Github/DS-strategy/current/
- **Документы стратегии:** ~/Github/DS-strategy/docs/ (Strategy.md, Dissatisfactions.md, Session Agenda.md)
- **Inbox:** ~/Github/DS-strategy/inbox/
- **SPOKE (планы репо):** ~/Github/*/WORKPLAN.md
- **MEMORY:** MEMORY.md (автозагрузка)

## Предусловие

> **Черновик WeekPlan уже создан** сценарием session-prep.
> Если черновика нет — сообщить пользователю и предложить запустить session-prep.

## Процесс

> Интерактивная сессия. Пользователь управляет через Claude Code.

#### 1. Открытие сессии

- Найди `WeekPlan W*.md` со `status: draft` в `DS-strategy/current/`
- Покажи сводку: повестка, ключевые вопросы, carry-over
- Спроси: «Начнём по повестке или есть срочные вопросы?»

#### 2. Ревью прошлой недели

- Пройди блок «Итоги» из черновика
- Обсуди: что удалось, что нет, почему

#### 3. Разбор inbox

- По каждому элементу: в план / capture / удалить / отложить
- Применяй решения сразу

#### 4. Неудовлетворённости (НЭП)

- Закрой разрешённые НЭП, обсуди новые
- Обнови Dissatisfactions.md

#### 5. Стратегическая сверка

- Обсуди расхождения стратегии и текущих РП
- Корректируй Strategy.md

#### 6. Формирование плана на неделю

- Пользователь корректирует РП, бюджеты, приоритеты

#### 7. Нерегулярные блоки (если есть)

- Пройди нерегулярные блоки из повестки

#### 8. Утверждение и синхронизация

- `status: draft` → `status: confirmed`
- Обнови WORKPLAN.md в целевых репо
- **ОБЯЗАТЕЛЬНО:** Обнови MEMORY.md → секция «РП текущей недели»
- **ОБЯЗАТЕЛЬНО:** Если добавлена работа вне стратегии → обнови Strategy.md
- **ОБЯЗАТЕЛЬНО:** Очисти обработанные inbox-заметки
- Закоммить

**Результат:** утверждённый WeekPlan, обновлённые Strategy.md, WORKPLAN.md, MEMORY.md, очищенный inbox.

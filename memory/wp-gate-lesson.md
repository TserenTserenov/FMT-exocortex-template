# Урок: WP Gate нельзя обходить самооправданием

## Что произошло

Агент самостоятельно квалифицировал задачу как «быстрый фикс» (п. 2.4) и начал работу без проверки списка РП и без ритуала согласования — хотя задача создавала артефакты и заняла существенно больше 15 минут.

## Почему это ошибка

- Задача создавала артефакты (файлы, скрипты)
- Тест: «Будет ли результат записан в файл? Да → нужен РП»
- Исключение «≤15 мин» не применимо к крупным задачам
- Агент сам себе разрешил исключение, вместо того чтобы следовать протоколу

## Правильная последовательность

1. Исследование (без артефактов) — это исключение, ОК
2. Когда пользователь просит ИСПРАВИТЬ → это уже создание артефактов → WP Gate
3. СТОП → показать таблицу РП → предложить формулировку → дождаться подтверждения
4. Только после записи РП → ритуал (Работа/РП/Метод) → работа

## Правило

**Переход от исследования к действию = переход через WP Gate.**
Даже если исследование было исключением, момент «теперь исправь» требует полной проверки.
Никогда не квалифицировать задачу как исключение самостоятельно, если результат — файлы.
